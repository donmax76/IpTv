# Структура проекта Remote Desktop

## Файлы проекта

### Основные файлы

- **remote_server.py** - Серверная часть программы
  - Захват экрана
  - Обработка команд мыши и клавиатуры
  - Файловый менеджер (просмотр дисков, скачивание/загрузка файлов)
  - Аутентификация по паролю

- **remote_client.py** - Клиентская часть с GUI
  - Интерфейс подключения
  - Просмотр удаленного рабочего стола
  - Файловый менеджер с навигацией
  - Скачивание и загрузка файлов

### Вспомогательные файлы

- **requirements.txt** - Список зависимостей Python
- **README.md** - Полная документация проекта
- **QUICK_START.md** - Быстрая инструкция по использованию
- **start_server.bat** - Скрипт для запуска сервера (Windows)
- **start_client.bat** - Скрипт для запуска клиента (Windows)

## Архитектура

### Протокол связи

1. **TCP сокеты** - базовое соединение
2. **JSON** - команды и метаданные
3. **Бинарные данные** - скриншоты и файлы
4. **Префиксы** - идентификация типа данных:
   - `SCREEN` - данные скриншота
   - `FILE_DATA` - данные файла
   - `FILE_END` - конец передачи файла

### Потоки выполнения

**Сервер:**
- Основной поток - прием подключений
- Поток отправки скриншотов
- Поток обработки команд от клиента

**Клиент:**
- Основной поток - GUI (tkinter)
- Поток приема данных от сервера

### Безопасность

- Аутентификация по паролю (SHA-256 хеш)
- Прямое P2P соединение (без промежуточных серверов)
- ⚠️ Данные передаются без шифрования (для продакшена требуется TLS/SSL)

## Зависимости

- **mss** - захват экрана
- **Pillow** - обработка изображений
- **pyautogui** - управление мышью и клавиатурой
- **pynput** - дополнительные возможности управления
- **opencv-python** - обработка изображений
- **numpy** - работа с массивами
- **pywin32** (опционально) - улучшенная работа с Windows

## Ограничения

- Только Windows (для полной поддержки)
- Нет шифрования данных
- Нет сжатия для больших файлов
- Одно подключение одновременно
- Качество изображения фиксированное
