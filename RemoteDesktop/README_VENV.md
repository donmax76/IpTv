# Виртуальное окружение - Интегрированные зависимости

## Что это?

Все зависимости проекта устанавливаются в **виртуальное окружение** (папка `venv`), которое находится прямо в проекте. Это означает:

✅ Зависимости интегрированы в проект  
✅ Не влияют на системный Python  
✅ Можно переносить проект вместе с зависимостями  
✅ Не нужно устанавливать зависимости глобально  

## Автоматическая установка

При первом запуске `start_server.bat` или `start_client.bat` автоматически:

1. Проверяется наличие виртуального окружения
2. Если его нет - создается автоматически
3. Устанавливаются все зависимости из `requirements.txt`
4. Запускается программа

**Вам ничего делать не нужно!** Просто запустите bat файл.

## Ручная установка

Если хотите установить зависимости вручную:

```bash
setup_venv.bat
```

Или:

```bash
install_dependencies.bat
```

## Структура проекта

```
RemoteDesktop/
├── venv/              ← Виртуальное окружение (создается автоматически)
│   ├── Scripts/       ← Исполняемые файлы
│   └── Lib/           ← Установленные пакеты
├── remote_server.py
├── remote_client.py
├── requirements.txt
├── setup_venv.bat     ← Создание venv
└── start_server.bat   ← Автоматически использует venv
```

## Преимущества

1. **Изоляция** - зависимости не конфликтуют с другими проектами
2. **Портативность** - можно скопировать весь проект с зависимостями
3. **Чистота** - системный Python остается чистым
4. **Простота** - автоматическая установка при первом запуске

## Размер

Виртуальное окружение занимает примерно 200-300 МБ после установки всех зависимостей.

## Удаление

Если нужно удалить виртуальное окружение:

```bash
rmdir /s /q venv
```

При следующем запуске оно создастся заново.

## Примечание

Папка `venv` добавлена в `.gitignore`, поэтому она не будет включена в git репозиторий. Каждый разработчик создает свое виртуальное окружение локально.
