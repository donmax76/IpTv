# Быстрый старт

## Установка

1. Установите Python 3.8 или выше
2. **Зависимости установятся автоматически в виртуальное окружение при первом запуске!**

   Все зависимости будут в папке `venv` внутри проекта - ничего не нужно устанавливать глобально.
   
   При первом запуске `start_server.bat` или `start_client.bat` автоматически:
   - Создастся виртуальное окружение
   - Установятся все зависимости
   - Запустится программа

## Использование

### Шаг 1: Запустите сервер на удаленном компьютере

**Вариант 1:** Двойной клик на `start_server.bat`

**Вариант 2:** Из командной строки:
```bash
python remote_server.py
```

**Важно:** Запишите IP адрес и пароль, которые отобразит сервер!

Пример вывода:
```
==================================================
Сервер запущен и ожидает подключения...
IP: 192.168.1.100
Порт: 5900
Пароль: abc123xyz
==================================================
```

### Шаг 2: Запустите клиент на вашем компьютере

**Вариант 1:** Двойной клик на `start_client.bat`

**Вариант 2:** Из командной строки:
```bash
python remote_client.py
```

### Шаг 3: Подключитесь

1. Введите IP адрес удаленного компьютера
2. Введите порт (по умолчанию 5900)
3. Введите пароль
4. Нажмите "Подключиться"

## Использование функций

### Удаленный рабочий стол

- **Просмотр экрана:** Вкладка "Рабочий стол" показывает экран удаленного компьютера
- **Управление мышью:** Кликните на экране для управления
- **Управление клавиатурой:** Нажмите клавиши на клавиатуре

### Файловый менеджер

1. Перейдите на вкладку "Файлы"
2. Нажмите "Диски" для просмотра всех дисков
3. Двойной клик по папке для открытия
4. Выберите файл и нажмите "Скачать" для скачивания
5. Нажмите "Загрузить" для загрузки файла на удаленный компьютер

## Решение проблем

### Не могу подключиться

1. Проверьте, что сервер запущен
2. Проверьте IP адрес (используйте локальный IP для сети, внешний для интернета)
3. Проверьте файрвол - должен разрешать входящие подключения на порт 5900
4. Проверьте пароль

### Медленная работа

- Уменьшите качество изображения в коде сервера (параметр `quality` в `_capture_screen`)
- Закройте другие программы, использующие сеть
- Используйте проводное соединение вместо Wi-Fi

### Ошибки с файлами

- Убедитесь, что у пользователя есть права на чтение/запись
- Проверьте путь к файлу
- Для больших файлов может потребоваться время

## Безопасность

⚠️ **Внимание:** Программа передает данные без шифрования. Используйте только в доверенных сетях или через VPN!

## Поддержка

При возникновении проблем проверьте:
1. Версию Python (должна быть 3.8+)
2. Установлены ли все зависимости
3. Логи сервера на наличие ошибок
